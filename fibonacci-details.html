1Ô∏è‚É£ Recursive (Naive) Method
O(2^n) Time
üí° Konsep Dasar

Metode paling sederhana yang mengikuti definisi matematika Fibonacci secara langsung. Setiap pemanggilan fungsi memanggil dirinya sendiri 2 kali untuk menghitung F(n-1) dan F(n-2).

üìù Pseudocode
function fibonacci(n): if n ‚â§ 1: return n return fibonacci(n-1) + fibonacci(n-2)
üîç Step-by-Step Example: F(5)
F(5) = F(4) + F(3)
F(4) = F(3) + F(2) ‚Üí F(3) dihitung lagi! ‚ùå
F(3) = F(2) + F(1)
F(2) = F(1) + F(0) = 1 + 0 = 1
F(3) = F(2) + F(1) ‚Üí F(2) dihitung lagi! ‚ùå

‚ö†Ô∏è Total Pemanggilan Fungsi: 15 kali untuk F(5)!
Banyak perhitungan yang sama diulang berkali-kali (redundant).

‚ö° Kompleksitas
Aspek Kompleksitas Penjelasan
Time O(2^n) Eksponensial! Setiap level membuat 2 cabang baru
Space O(n) Call stack depth maksimal = n
Praktis untuk n ‚â§ 40 (lebih dari itu sangat lambat)
‚öñÔ∏è Kelebihan & Kekurangan

‚úÖ Kelebihan

  • Sangat mudah dipahami
  • Kode sangat singkat (3 baris)
  • Sesuai definisi matematika
  • Bagus untuk pembelajaran konsep rekursi

‚ùå Kekurangan

  • Sangat lambat untuk n besar
  • Banyak perhitungan redundan
  • Stack overflow risk untuk n sangat besar
  • Tidak praktis untuk produksi
üéØ Use Case

Kapan menggunakan: Hanya untuk pembelajaran, demonstrasi konsep rekursi, atau n sangat kecil (< 30).
Jangan gunakan: Untuk aplikasi nyata atau n > 35.

5Ô∏è‚É£ OpenMP Parallelization
Parallel Computing
üí° Konsep Dasar

OpenMP (Open Multi-Processing) adalah API untuk shared-memory parallel computing. Untuk Fibonacci, kita gunakan recursive parallelization: ketika menghitung F(n), kita jalankan F(n-1) dan F(n-2) di thread berbeda secara bersamaan!

üìù Pseudocode dengan OpenMP
#pragma omp parallel sections { // Task 1: Hitung F(n-1) di thread 1 #pragma omp section { int f1 = fibonacci(n - 1); } // Task 2: Hitung F(n-2) di thread 2 #pragma omp section { int f2 = fibonacci(n - 2); } } return f1 + f2;
üîç Execution Timeline: F(5) dengan 2 Threads
SERIAL (1 thread):
Calculate F(5) ‚Üí F(4) ‚Üí F(3) ‚Üí F(2) ‚Üí F(1) ‚Üí Done
‚è±Ô∏è Total: 15ms (sequential)
PARALLEL (2 threads):
Time 0ms: Start F(5)
Thread 1: F(4) starting...
Thread 2: F(3) starting... (parallel!)
Time 8ms: F(3) done, back to main
Thread 1: Still computing F(4)...
Time 12ms: F(4) done, sync point
Main: F(5) = F(4) + F(3) = Done
‚è±Ô∏è Total: 12ms (1.25x faster with 2 threads)
‚ö° Performance Characteristics
Faktor Penjelasan
Speedup Tergantung pada n dan jumlah core CPU
Contoh: F(35) dengan 8 cores: 3.36x faster
Thread Overhead Ada overhead untuk create/spawn threads
Tidak efficient untuk n kecil (n < 30)
Cutoff Strategy Gunakan serial execution untuk n < CUTOFF (misal 20)
Ini mengurangi thread overhead
Best Result F(35) + CUTOFF=20 + 8 threads: 3.6x - 3.8x speedup
‚öñÔ∏è Kelebihan & Kekurangan

‚úÖ Kelebihan

  • Memanfaatkan multi-core CPU
  • Significant speedup untuk n besar
  • Relatively simple implementation
  • Widely supported (gcc, clang, icc)
  • Production-ready solution

‚ùå Kekurangan

  • Thread creation overhead
  • Cutoff threshold harus di-tune
  • Memory bandwidth becomes bottleneck
  • Not efficient untuk n < 30
  • Performance varies by system
üéØ Practical Usage

Compile: gcc -fopenmp -O3 fib.c -o fib
Run: ./fib 35 (uses all available cores)
Control threads: OMP_NUM_THREADS=4 ./fib 35
Best for: Production systems, scientific computing, real-world applications

6Ô∏è‚É£ Cilk Parallelization (OpenCilk)
Task Parallelism
üí° Konsep Dasar

Cilk (C with Intellegent Language Kits) adalah bahasa extension untuk parallel programming yang lebih elegant dari OpenMP. Dengan keywords cilk_spawn dan cilk_sync, kita bisa dengan mudah membuat task parallelism. OpenCilk adalah fork terbaru yang lebih optimal.

üìù Pseudocode dengan Cilk
cilk_int fibonacci(n) { if (n ‚â§ 1) return n; cilk_spawn f1 = fibonacci(n - 1); // Spawn task untuk F(n-1) - jalankan di background int f2 = fibonacci(n - 2); // Continue dengan F(n-2) di current thread cilk_sync; // Wait untuk f1 selesai return f1 + f2; }
üîç Cilk vs OpenMP Comparison
Aspek OpenMP Cilk
Syntax #pragma omp parallel (directive) cilk_spawn (keyword)
Granularity Control Manual (programmer controls) Automatic (Cilk scheduler optimizes)
Work Stealing Not built-in Yes! Smart load balancing
Scheduling Static/dynamic (user specified) Work-stealing scheduler
Code Clarity Pragmas can clutter code More natural & readable
Performance Good with proper tuning Better auto-tuning (usually faster)
Community Very large, widely used Smaller but growing (MIT, Intel)
‚öôÔ∏è How Cilk Work Stealing Scheduler Works

Concept: Jika sebuah thread selesai lebih dulu, ia akan "mencuri" pekerjaan dari thread lain yang masih sibuk.

  1. Initial: F(10) di-spawn ke 4 threads
  2. Thread 1: Mengerjakan F(10) ‚Üí mendapat banyak sub-tasks
  3. Thread 2: Selesai cepat, steal task dari Thread 1's deque
  4. Thread 3, 4: Juga steal tasks yang ada
  5. Result: Load balancing otomatis & optimal!

‚úÖ Ini membuat Cilk lebih efisien dari OpenMP untuk recursive parallelism!

‚öñÔ∏è Kelebihan & Kekurangan

‚úÖ Kelebihan

  • Elegant syntax dengan spawn/sync
  • Work-stealing scheduler (auto load balancing)
  • Better for recursive parallelism
  • Less manual tuning needed
  • Proven better performance (MIT research)
  • More intuitive for divide & conquer

‚ùå Kekurangan

  • Smaller community than OpenMP
  • Less widely supported (needs special compiler)
  • OpenCilk is relatively new
  • Learning curve for work-stealing concept
  • Availability on some systems limited
üéØ Practical Usage

Install OpenCilk: brew install opencilk (macOS)
Compile: clang -fopencilk -O3 fib.cilk.c -o fib
Run: ./fib 35
Optimal for: Recursive algorithms, tree traversal, divide & conquer problems

üéì Algorithm Selection Guide

  • üìö Learning Recursion: Start with Recursive method (simple and intuitive)
  • ‚ö° Parallel Computing (OpenMP): Multi-threaded version with manual tuning (3.6x-3.8x speedup on 8 cores)
  • ‚ö°‚ö° Advanced Parallel (Cilk): Task-based with automatic work-stealing scheduler (better load balancing)

üìå Key Insights

  • All 3 methods use recursive approach: Calculate F(n) = F(n-1) + F(n-2)
  • Key Difference: Execution strategy - Sequential, OpenMP (manual threading), or Cilk (automatic work-stealing)
  • Parallel Speedup: Limited by Amdahl's Law: Speedup = 1 / (1 - p + p/n), where p = parallelizable part
  • Cutoff Strategy: Both OpenMP and Cilk use serial base case for small n (< 20) to reduce overhead
  • Work Stealing (Cilk): Automatic load balancing - when a thread finishes early, it "steals" work from busy threads
