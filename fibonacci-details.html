1Ô∏è‚É£ Recursive (Naive) Method
O(2^n) Time
üí° Konsep Dasar

Metode paling sederhana yang mengikuti definisi matematika Fibonacci secara langsung. Setiap pemanggilan fungsi memanggil dirinya sendiri 2 kali untuk menghitung F(n-1) dan F(n-2).

üìù Pseudocode
function fibonacci(n): if n ‚â§ 1: return n return fibonacci(n-1) + fibonacci(n-2)
üîç Step-by-Step Example: F(5)
F(5) = F(4) + F(3)
F(4) = F(3) + F(2) ‚Üí F(3) dihitung lagi! ‚ùå
F(3) = F(2) + F(1)
F(2) = F(1) + F(0) = 1 + 0 = 1
F(3) = F(2) + F(1) ‚Üí F(2) dihitung lagi! ‚ùå

‚ö†Ô∏è Total Pemanggilan Fungsi: 15 kali untuk F(5)!
Banyak perhitungan yang sama diulang berkali-kali (redundant).

‚ö° Kompleksitas
Aspek Kompleksitas Penjelasan
Time O(2^n) Eksponensial! Setiap level membuat 2 cabang baru
Space O(n) Call stack depth maksimal = n
Praktis untuk n ‚â§ 40 (lebih dari itu sangat lambat)
‚öñÔ∏è Kelebihan & Kekurangan

‚úÖ Kelebihan

  • Sangat mudah dipahami
  • Kode sangat singkat (3 baris)
  • Sesuai definisi matematika
  • Bagus untuk pembelajaran konsep rekursi

‚ùå Kekurangan

  • Sangat lambat untuk n besar
  • Banyak perhitungan redundan
  • Stack overflow risk untuk n sangat besar
  • Tidak praktis untuk produksi
üéØ Use Case

Kapan menggunakan: Hanya untuk pembelajaran, demonstrasi konsep rekursi, atau n sangat kecil (< 30).
Jangan gunakan: Untuk aplikasi nyata atau n > 35.

2Ô∏è‚É£ Dynamic Programming (Bottom-Up)
O(n) Time, O(n) Space
üí° Konsep Dasar

Menyimpan semua nilai Fibonacci dari 0 hingga n dalam sebuah array. Setiap nilai dihitung HANYA SEKALI dan disimpan untuk digunakan kembali. Pendekatan bottom-up: mulai dari F(0), F(1), lalu naik ke F(n).

üìù Pseudocode
function fibonacci(n): if n ‚â§ 1: return n // Buat array untuk menyimpan hasil dp = [0, 1] for i from 2 to n: // F(i) = F(i-1) + F(i-2) dp[i] = dp[i-1] + dp[i-2] return dp[n]
üîç Step-by-Step Example: F(7)
Inisialisasi: dp = [0, 1]
i=2: dp[2] = dp[1] + dp[0] = 1 + 0 = 1 ‚Üí [0, 1, 1]
i=3: dp[3] = dp[2] + dp[1] = 1 + 1 = 2 ‚Üí [0, 1, 1, 2]
i=4: dp[4] = dp[3] + dp[2] = 2 + 1 = 3 ‚Üí [0, 1, 1, 2, 3]
i=5: dp[5] = dp[4] + dp[3] = 3 + 2 = 5 ‚Üí [0, 1, 1, 2, 3, 5]
i=6: dp[6] = dp[5] + dp[4] = 5 + 3 = 8 ‚Üí [0, 1, 1, 2, 3, 5, 8]
i=7: dp[7] = dp[6] + dp[5] = 8 + 5 = 13 ‚Üí [0, 1, 1, 2, 3, 5, 8, 13]

‚úÖ Total Operasi: 7 iterasi saja!
Tidak ada perhitungan redundan - setiap nilai dihitung tepat 1 kali.

‚ö° Kompleksitas
Aspek Kompleksitas Penjelasan
Time O(n) Linear! Loop dari 2 sampai n
Space O(n) Array menyimpan n+1 elemen
Praktis untuk n ‚â§ 10,000+ (sangat cepat)
‚öñÔ∏è Kelebihan & Kekurangan

‚úÖ Kelebihan

  • Sangat cepat - O(n) time
  • Mudah dipahami (iterative)
  • Bisa akses semua nilai kapan saja
  • Bagus untuk print semua sequence
  • Praktis untuk produksi

‚ùå Kekurangan

  • Butuh O(n) memory
  • Memory waste jika hanya butuh F(n)
  • Tidak optimal untuk n sangat besar
üéØ Use Case

Kapan menggunakan: Ketika butuh semua nilai Fibonacci dari 0 sampai n, atau butuh print sequence.
Perfect untuk: Aplikasi yang perlu menyimpan history, visualisasi sequence, atau lookup table.

3Ô∏è‚É£ Space Optimized (Iterative)
O(n) Time, O(1) Space
üí° Konsep Dasar

Optimisasi dari DP dengan observasi: untuk menghitung F(n), kita hanya butuh F(n-1) dan F(n-2). Tidak perlu simpan SEMUA nilai! Cukup simpan 2 variabel terakhir saja. Ini adalah trade-off: korbankan akses ke semua nilai, dapat memory efficiency.

üìù Pseudocode
function fibonacci(n): if n ‚â§ 1: return n // Hanya 2 variabel! prev = 0 curr = 1 for i from 2 to n: // Hitung nilai baru next = prev + curr // Shift: prev ‚Üí curr ‚Üí next prev = curr curr = next return curr
üîç Step-by-Step Example: F(7)
Inisialisasi: prev=0, curr=1
i=2: next = 0+1=1 ‚Üí prev=1, curr=1
i=3: next = 1+1=2 ‚Üí prev=1, curr=2
i=4: next = 1+2=3 ‚Üí prev=2, curr=3
i=5: next = 2+3=5 ‚Üí prev=3, curr=5
i=6: next = 3+5=8 ‚Üí prev=5, curr=8
i=7: next = 5+8=13 ‚Üí prev=8, curr=13

‚úÖ Total Memory: HANYA 2 variabel (prev & curr)!
Speed sama dengan DP, tapi memory jauh lebih hemat.

‚ö° Kompleksitas
Aspek Kompleksitas Penjelasan
Time O(n) Linear - sama dengan DP
Space O(1) Constant! Hanya 2-3 variabel
Praktis untuk n ‚â§ 1,000,000+ (super efficient)
‚öñÔ∏è Kelebihan & Kekurangan

‚úÖ Kelebihan

  • Paling hemat memory - O(1)!
  • Tetap cepat - O(n) time
  • Simple & elegant
  • Perfect untuk production
  • Bisa handle n sangat besar

‚ùå Kekurangan

  • Hanya dapat nilai akhir F(n)
  • Tidak bisa akses F(i) lainnya
  • Tidak cocok jika butuh sequence
üéØ Use Case

Kapan menggunakan: Ketika hanya butuh F(n) saja (tidak butuh semua sequence).
Perfect untuk: Production code, embedded systems, atau ketika memory sangat terbatas.
Best Choice: Untuk sebagian besar aplikasi real-world!

4Ô∏è‚É£ Matrix Exponentiation
O(log n) Time
üí° Konsep Dasar

Metode paling advanced yang menggunakan aljabar linear! Berdasarkan fakta bahwa: [F(n+1), F(n)] = [F(1), F(0)] √ó [[1,1],[1,0]]^n
Dengan fast matrix exponentiation (divide & conquer), kita bisa hitung pangkat dalam O(log n)!

üìù Pseudocode
function fibonacci(n): if n ‚â§ 1: return n // Base matrix untuk Fibonacci baseMatrix = [[1, 1], [1, 0]] // Hitung matrix^n dengan fast exponentiation resultMatrix = matrixPower(baseMatrix, n) return resultMatrix[0][1] function matrixPower(M, n): if n == 1: return M if n is even: // M^n = (M^(n/2))^2 half = matrixPower(M, n/2) return matrixMultiply(half, half) else: // M^n = M √ó M^(n-1) return matrixMultiply(M, matrixPower(M, n-1))
üîç Step-by-Step Example: F(6)
Target: Hitung [[1,1],[1,0]]^6
Langkah 1: M^6 = (M^3)^2 ‚Üí Hitung M^3 dulu
M^3 = M √ó M^2 ‚Üí Hitung M^2 dulu
M^2 = M √ó M = [[1,1],[1,0]] √ó [[1,1],[1,0]] = [[2,1],[1,1]]
M^3 = M^2 √ó M = [[2,1],[1,1]] √ó [[1,1],[1,0]] = [[3,2],[2,1]]
Langkah 2: M^6 = M^3 √ó M^3 = [[3,2],[2,1]] √ó [[3,2],[2,1]] = [[13,8],[8,5]]
Hasil: F(6) = resultMatrix[0][1] = 8 ‚úÖ

‚úÖ Total Perkalian Matrix: Hanya 3 kali untuk F(6)!
Untuk F(1000): Hanya ~10 perkalian matrix (log‚ÇÇ 1000 ‚âà 10)

üßÆ Kenapa O(log n)?

Fast exponentiation bekerja dengan membagi eksponen jadi 2 setiap kali:

  • M^32 = (M^16)^2 = ((M^8)^2)^2 = ...
  • Pangkat 32 hanya butuh 5 langkah (2^5 = 32)
  • Pangkat 1024 hanya butuh 10 langkah (2^10 = 1024)
  • Secara umum: n langkah ‚Üí log‚ÇÇ n operasi
‚ö° Kompleksitas
Aspek Kompleksitas Penjelasan
Time O(log n) Logaritmik! Tercepat secara asimptotik
Space O(log n) Recursion call stack depth = log n
Praktis untuk n sangat besar (10^6, 10^9, dst)
‚öñÔ∏è Kelebihan & Kekurangan

‚úÖ Kelebihan

  • TERCEPAT untuk n besar - O(log n)!
  • Elegant secara matematis
  • Bisa handle n = 10^9+
  • Impressive untuk competitive programming

‚ùå Kekurangan

  • Butuh pemahaman aljabar linear

  • Tidak intuitive untuk pemula
üéØ Use Case

Kapan menggunakan: Ketika n sangat besar (> 10^6), atau untuk competitive programming.
Perfect untuk: Contests, challenges, atau untuk show off mathematical elegance! üòé
Catatan: Untuk n < 1000, Space Optimized lebih praktis dan sama cepatnya.
