üìñ Apa itu Bilinear Interpolation?
Image Processing

Bilinear Interpolation adalah teknik matematika untuk mengestimasi nilai pada titik mana pun di dalam area 2D berdasarkan nilai 4 titik corner di sekitarnya. Teknik ini sangat penting dalam image processing, terutama untuk image resizing, rotation, dan texture mapping dalam computer graphics.

üéØ Prinsip Dasar:
Interpolasi dilakukan dalam 2 tahap:
  1. Linear Interpolation di arah X - Interpolasi horizontal antara corner kiri dan kanan
  2. Linear Interpolation di arah Y - Interpolasi vertikal dari hasil step 1
üí° Kenapa "Bi-linear"?
"Bi" = Dua ‚Üí Linear interpolation dilakukan 2 kali (sekali untuk X, sekali untuk Y). Makanya disebut "Bi-linear" bukan cuma "linear"!
üõ†Ô∏è Cara Kerja Praktis
Mapping ‚Üí Neighbor ‚Üí Fraction

1) Mapping Koordinat (dest ‚Üí src)

Rumus:
srcX = destX √ó (srcWidth / destWidth)
srcY = destY √ó (srcHeight / destHeight)

Contoh: resize 100√ó100 ‚Üí 200√ó200, piksel (150,120) di output dipetakan ke (75.0, 60.0) di gambar asli. Koordinat ini biasanya non-integer ‚Üí perlu interpolasi.

2) Cari 4 Pixel Tetangga

Gunakan floor/ceil:
x0 = floor(srcX), x1 = ceil(srcX)
y0 = floor(srcY), y1 = ceil(srcY)
Ambil f00 (x0,y0), f10 (x1,y0), f01 (x0,y1), f11 (x1,y1).

3) Hitung Fractional Part

Fraksi jarak:
fx = srcX - x0
fy = srcY - y0

Pakai fx, fy pada rumus utama: f(x,y) = (1-fx)(1-fy)f00 + fx(1-fy)f10 + (1-fx)fy f01 + fx fy f11.
Contoh cepat (sesuai demo):
- Gambar 100√ó100 di-resize ke 200√ó200 ‚áí piksel output (150,120) dipetakan ke (75.0,60.0) di sumber.
- Tetangga: x0=75, x1=75, y0=60, y1=60 (jika tepat integer, fx=fy=0 ‚áí nilai persis f00).
- Jika koordinat non-integer (mis. 75.3, 60.7) ‚áí fx=0.3, fy=0.7 dan rumus bilinear dipakai pada f00,f10,f01,f11.
- Inilah urutan yang dipakai backend C (OpenMP) di demo: mapping ‚Üí ambil 4 tetangga ‚Üí hitung fx,fy ‚Üí kombinasi bilinear.

Ini memastikan hasil interpolasi berada di range yang sama dengan input values!
üé® Bilinear Interpolation untuk Image Upscaling
4 Tetangga Terdekat
Inti: Setiap pixel baru dihitung dari 4 pixel terdekat di gambar asli (Q11, Q21, Q12, Q22). Bobot ditentukan oleh jarak fraksional (fx, fy) dari koordinat hasil mapping dest‚Üísrc.

üîÅ Pipeline Upscaling (dipakai di demo/index.html)

  • Mapping: srcX = destX √ó (srcW/destW), srcY = destY √ó (srcH/destH).
  • Ambil 4 tetangga: Q11 (floor, floor), Q21 (ceil, floor), Q12 (floor, ceil), Q22 (ceil, ceil).
  • Hitung fraksi: fx = srcX - floor(srcX), fy = srcY - floor(srcY).
  • Blend bilinear: (1-fx)(1-fy)Q11 + fx(1-fy)Q21 + (1-fx)fy Q12 + fx fy Q22.

‚ö° Kenapa Dipilih untuk Demo?

  • Hasil lebih halus dibanding nearest neighbor; cukup ringan dibanding bicubic.
  • Kompleksitas O(1) per pixel, cocok untuk real-time atau batch ringan.
  • Memory-bound: speedup paralel dibatasi bandwidth memori.
  • Mudah diparalelisasi dengan OpenMP (tiap pixel independen).
‚öñÔ∏è Kelebihan & Kekurangan Bilinear
Analysis

‚úÖ Kelebihan

  • Balance sempurna antara speed & quality
  • Mudah diimplementasikan
  • Kompleksitas O(1) per pixel
  • Hasil smooth & natural untuk kebanyakan kasus
  • Hardware-accelerated di GPU
  • Widely supported di semua platform

‚ùå Kekurangan

  • Slight blur pada edges tajam
  • Tidak sebaik bicubic untuk detail halus
  • Bisa ada aliasing artifacts untuk downscaling besar
  • Tidak preserve edges dengan sempurna
üí° Kesimpulan:
Bilinear adalah "sweet spot" untuk 90% use case! Cukup cepat untuk real-time processing, cukup bagus untuk sebagian besar aplikasi. Gunakan bicubic/lanczos hanya jika butuh kualitas maksimal dan punya waktu untuk komputasi lebih lama.
