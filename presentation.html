Fibonacci - Metode

Sequential Baseline

  • Rekursif klasik: F(n)=F(n-1)+F(n-2)
  • Satu thread, overhead 0%
  • Digunakan sebagai acuan speedup

OpenMP Task-based

  • Task untuk F(n-1); thread aktif kerjakan F(n-2)
  • Cutoff n < 20 dieksekusi sequential untuk hindari overhead
  • Metrik: speedup & efficiency

OpenCilk Work-Stealing

  • cilk_spawn untuk F(n-1), cilk_sync gabung hasil
  • Scheduler work-stealing: worker idle mencuri task
  • Cutoff sama: sequential untuk n kecil

Bilinear Interpolation - Inti

  • Rumus: f(x,y) = (1-fx)(1-fy)f00 + fx(1-fy)f10 + (1-fx)fy f01 + fx fy f11
  • Mapping dest→src: srcX = destX × (srcW/destW), srcY = destY × (srcH/destH)
  • Tetangga: f00, f10, f01, f11 (floor/ceil koordinat)
  • Fractional part: fx = srcX - floor(srcX), fy = srcY - floor(srcY)
  • Kompleksitas O(1) per piksel → sangat cepat

Tooling & Build

Compile
  • gcc-15 -fopenmp -DUSE_OPENMP -o fib_omp_json fibonacci_json.c
  • <opencilk-clang> -fopencilk -DUSE_CILK -o fib_json_cilk fibonacci_json.c
  • gcc-15 -fopenmp -O3 -o bilinear bilinear_serial_parallel.c -lm
Server
  • Node.js + Express
  • Endpoints: /api/fibonacci/:n?mode=, /api/bilinear
  • Static: index.html, halaman detail
UI
  • Fibonacci cards: sequential, OpenMP serial/parallel, Cilk serial/parallel
  • Bilinear: info ukuran, hasil serial/parallel, perbandingan gambar

Kesimpulan

  • Fibonacci: parallel tasking (OpenMP/Cilk) memberi speedup signifikan untuk n besar; cutoff penting untuk kontrol overhead.
  • Bilinear: pipeline mapping → tetangga → fx,fy → kombinasi; OpenMP mempercepat tetapi dibatasi bandwidth memori.
  • Stack lengkap: C backend + Node API + UI interaktif memudahkan eksplorasi performa.
